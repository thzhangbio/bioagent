### 结论：文献原文 vs 文献向量库，哪个更好

结论不是简单二选一，而是分角色使用：

- 默认生产生成时，优先使用“文献向量化后的 `literature` 库”作为事实来源。
- 若用户明确要求仿梅斯学术，则同时调用 `wechat_style` 中的 `medsci` 子风格作为表达参考。
- 文献原文更适合作为审计、抽查、补证和疑难回溯材料，而不应作为每次生成时直接整篇喂给模型的默认输入。

### 原因分析

#### 1. 向量化后的文献库更适合生产生成

- 现有代码已经围绕 `literature` 做了检索与命中验证，能先确认“这篇文献在库里确实存在并命中”，再进入生成。
- 向量检索能只取与当前任务最相关的事实片段，减少整篇原文过长、噪声过多、模型注意力分散的问题。
- 现有分层设计明确要求：事实与风格分开，避免把风格语料误当事实来源；这与当前双库方案完全一致。
- 现成代码路径最短，复用性最好，后续也最容易扩展到更多子风格和槽位。

#### 2. 文献原文不适合作为默认生产输入

- 文献原文通常很长，包含作者信息、页眉页脚、方法学细节、图表残留、版面噪声等，会稀释模型对“核心研究结论”的注意力。
- 即使清洗过，整篇直接投喂仍然比“命中的关键片段”更耗 token，也更容易把非重点段落带进生成。
- 当任务是“仿梅斯学术写公众号长文”时，模型需要的不是全文复述，而是“抓核心研究信息 + 套目标风格结构”；这更适合检索增强而非整篇硬塞。

#### 3. 文献原文仍然有价值

- 当向量检索命中不足、片段缺失、需要核对具体数字/纳排标准/方法学细节时，原文应作为回溯依据。
- 更适合在以下场景启用：
  - 抽查模型是否忠于原文
  - 某篇文献生成结果需要人工复核
  - 向量切块不完整，需补充关键上下文
  - 后续要做引用追溯、页码级核对

### 推荐的默认策略

- 事实层：`literature`
- 风格层：`wechat_style` 且限定 `medsci`
- 原文：仅作为补充校验材料，不直接作为每次生成的默认主输入

换句话说：

- 最优默认方案：`literature` + `wechat_style(medsci)`
- 不推荐默认方案：直接整篇文献原文 + 梅斯风格提示词
- 更不推荐方案：只用梅斯风格库，不接文献事实库

### 对“只调用梅斯学术库不一定有现成代码”的判断

- 现有代码里，成熟路径是“文献命中验证 + 文献事实块 + 梅斯风格块”一起进入生成。
- 只调用梅斯学术库，从产品语义上也不合理，因为它只能提供表达习惯，不能提供可靠事实。
- 即使技术上后续补一个“只用 `wechat_style`”的入口，也不适合作为文献改写任务的正式生产方案。

#### 总体风格规律

##### 1. 开篇风格

39 篇样本中，开头最稳定的模式不是“论文摘要改写”，而是“问题化导入 + 认知反差 + 研究落点”：

- 约 8/39 的首句直接以问句起手
- 约 29/39 在前 1-3 段内出现明显转折词或认知落差词，如：
  - `然而`
  - `但`
  - `不过`
  - `却`
  - `不是`
  - `真正`
  - `意想不到`
- 约 33/39 在前 3 段内明确落下“机构 / 团队 / 期刊 / 研究”信息
- 约 31/39 在前 3 段内直接给出研究发现或主结论的预告

也就是说，梅斯学术的开篇通常遵循：

1. 先把读者带进一个熟悉问题、临床困惑或生活场景  
2. 立刻制造“原来不是你以为的那样”的认知反差  
3. 很快点出“谁做的研究、发在哪、核心结论是什么”  

它不是等到正文后半才讲发现，而是会在开头就先“剧透”半步，让读者知道这篇文章值得继续读。

##### 2. 分节小标题风格

梅斯学术的分节标题并不统一，但有两个明显谱系：

- 学术综述型：
  - `一、研究背景`
  - `二、方法与结果`
  - `三、总结`
- 公众号表达型：
  - `10%，不是终点，可能只是起点`
  - `它不是一种病，而是一群病`
  - `为什么分子诊断对儿童狼疮如此重要？`

整体上，小标题更像“一个明确判断 / 一个问题 / 一个结论句”，而不是教科书式目录。

经验上更接近以下特征：

- 尽量短
- 尽量有立场
- 尽量可传播
- 标题本身就能传达这一段的主旨

##### 3. 结尾收束方式

梅斯学术的正文结尾，核心不是“温和结束”，而是“再把主判断收紧一次”。

常见收束动作包括：

- 用 `综上所述` / `总而言之` / `小结` / `结语` 拉回主线
- 再次强调研究对临床、机制理解或方法学路径的意义
- 常把“这意味着什么”单独说透，而不仅仅重复结果

归档稿尾部还会固定接：

- `参考资料 / 文献`
- `署名·撰文 / 编辑`
- `转载说明`
- `联系方式`
- `运营导流`

需要注意的是：

- 这类尾部运营模块属于公众号成品结构
- 不是模型正文生成本身最核心的风格点
- 真正影响“像不像梅斯”的，仍然是开头推进方式、段中判断句密度和结尾收束力度

#### 更细的行文风格特征

##### 1. 句子节奏

- 常用短句 + 中长句交替
- 高频使用破折式解释、冒号后判断、括号补充
- 喜欢在一段中先给判断，再给解释，而不是一直铺背景

##### 2. 立场感

- 梅斯学术不是纯中性摘要
- 经常出现带判断色彩的话语，如：
  - `真正的关键是`
  - `这意味着`
  - `这一发现有力说明`
  - `这一结果提示`
  - `并不只是`
  - `不是一种病，而是一群病`

##### 3. 临床转译意识

- 不是只讲“研究发现了什么”
- 还会不断回答：
  - 对医生意味着什么
  - 对诊断路径意味着什么
  - 对治疗决策意味着什么
  - 对既有认知意味着什么

#### 可用于修正偏差的提示词方向

以下提示词不要求完全逐字照搬，而是建议作为生成约束补充到 prompt 中。

##### 1. 开篇修正提示词

- 开头不要直接按论文摘要顺序复述，请先从临床困惑、认知误区或高频问题切入
- 前 2 段内必须制造一次明确认知反差，例如“原来不是……而是……”“真正的问题在于……”
- 前 3 段内明确交代研究团队、期刊层级与核心发现，但不要写成机械摘要
- 开头要让读者迅速感到“这项研究正在改写原有理解”

##### 2. 小标题修正提示词

- 小标题不要只写“研究背景/研究结果”，优先写成带判断的短句型标题
- 每个二级标题都要能单独表达该段主结论
- 标题优先采用“结论句 / 问句 / 认知反差句”，减少教科书式目录感

##### 3. 正文推进修正提示词

- 每一节先给结论，再展开数据和机制解释
- 每 1-2 段至少出现一次“这意味着 / 这一发现提示 / 真正关键在于 / 这一结果有力说明”式判断句
- 不要只解释研究做了什么，更要持续回答“对临床意味着什么”

##### 4. 结尾修正提示词

- 结尾不要只做总结，要用一句高度压缩的判断句完成收束
- 最后一段要明确指出这项研究改变了什么认知、会推动什么临床路径或方法学转向
- 让读者读完后能记住一个清晰结论，而不是只记住“做了全外显子测序”

#### 当前结论

- 03 这篇已经不是“差稿”，而是“有梅斯骨架、但判断力和推进张力还不够”的稿件
- 最需要修正的，不是事实层，而是：
  - 开头的认知落差强度
  - 中段判断句密度
  - 结尾的压缩收束能力
- 后续若要继续提升生成质量，优先应在 prompt 中加强“先判断、后展开”的生成约束，而不是继续堆更多事实细节
