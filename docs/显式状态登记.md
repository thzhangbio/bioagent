# 显式状态登记

> **目的**：集中说明仓库内可配置的**显式状态**（环境变量含义、侧车 CLI、灌库行为、`data/` 文件等），避免散落在多份文档中难以查找。  
> **与代码一致的快照**：默认 RAG 集合、`.env.example` 变量名列表、本机 `rag-store.json` 各集合块数、Embedding 默认模型等，由脚本从代码与向量文件**自动生成**，见 **`显式状态.generated.md`**。生成命令：`pnpm run report:explicit-state`（改集合常量或灌库后请重跑，便于 PR/文档与真实状态对齐）。  
> **维护**：本页保留**语义说明**；变更开关或路径时，请更新下文相关段落，并**重跑**上述命令刷新生成文件。下方「修订记录」仅记文档层面的重要调整。

## 一、环境变量（`.env` / `.env.example`）

**变量名清单**（与仓库约定同步）：见 **`显式状态.generated.md` §6**。下表为各变量**含义**说明（手维护，与实现一致时请以此为准）。

| 变量 | 作用与生效范围 |
|------|----------------|
| **`FEISHU_APP_ID`** / **`FEISHU_APP_SECRET`** | 飞书应用凭证；**长连接事件**、OpenAPI（云文档、IM 等）均需。**必填**（`src/index.ts` 启动校验）。 |
| **`FEISHU_WEB_BASE`** | 生成云文档浏览器链接时的站点根；默认 `https://feishu.cn`；自建/国际版可改。 |
| **`FEISHU_DOC_FOLDER_TOKEN`** | 新建云文档时的父目录（云空间文件夹 token）；不设则落在应用默认位置。 |
| **`FEISHU_DOC_DISABLE_USER_GRANT`** | 设为 `1`：**跳过**创建 docx 后给发消息用户加「可管理」协作者（调试）。 |
| **`FEISHU_CONTENT_FORCE_GATE`** | 设为 `1`：**关闭**写作简报触发的强制通过闸门（排障）；见 `write-brief.ts`。 |
| **`FEISHU_API_MAX_RETRIES`** | 飞书 OpenAPI TLS/断连重试次数；默认 `4`（`docx-document.ts`）。 |
| **`FEISHU_CONTENT_CARD_FORM`** | 设为 `1`：内容创作意图先下发**交互卡片**确认，再进工作流；需订阅 `card.action.trigger` 等（`index.ts`、`events.ts`）。 |
| **`ANTHROPIC_API_KEY`** | Claude（经 APIMart 等）调用密钥。 |
| **`ANTHROPIC_BASE_URL`** | 中转站 Base URL；不设则用 SDK 默认。 |
| **`ANTHROPIC_MODEL`** | 模型 ID；未设时代码默认 `claude-sonnet-4-6`（`router.ts`、`content-create.ts` 等）。 |
| **`CHAT_WINDOW_MAX_MESSAGES`** | **可选**：会话滑动窗口最大消息条数（user+assistant 各算一条）；超出则并入滚动摘要；默认 `24`（`chat-session.ts`）。 |
| **`CHAT_WINDOW_TOTAL_CHARS`** | **可选**：窗口内消息总字符软上限；超出则触发摘要压缩；默认 `48000`。 |
| **`CHAT_SUMMARY_MAX_CHARS`** | **可选**：滚动摘要硬截断长度；默认 `2500`。 |
| **`OPENAI_API_KEY`** | **Embedding** 与用户上传文件入库；与 Claude 可共用同一 APIMart sk。 |
| **`OPENAI_BASE_URL`** | Embeddings 兼容端点；默认 APIMart `/v1`。 |
| **`EMBEDDING_MODEL`** | 默认 `text-embedding-3-large`（`embeddings.ts`）。 |
| **`DEBUG_RAG`** | 设为 `1`：打印 RAG 检索/入库调试信息（勿长期开）。 |
| **`DEBUG_LARK`** | 设为 `1`：飞书 SDK 日志更细（`events.ts`、`content-create-card-flow.ts` 等）。 |
| **`LITERATURE_INBOX`** | **`pnpm run ingest:literature`**：覆盖文献 inbox 目录（相对仓库根或绝对路径）。 |
| **`WECHAT_MP_COOKIE`** | 微信侧车 **`--fetch-stats`**：请求 `getappmsgext` 用的 Cookie（无则互动数常为空）。 |
| **`WECHAT_STYLE_INBOX`** | **`pnpm run ingest:wechat`**：覆盖微信成稿目录；默认 `side-tools/wechat-article-cleanup/out/`。 |
| **`WECHAT_DEFAULT_VARIANT`** | **`ingest:wechat`**：当 md 无 `wechat_style_variant` 且无 `.meta.json` 时子风格；默认 `medsci`。 |

## 二、RAG：默认检索集合（以生成快照为准）

**单一数据源**：`src/knowledge/rag-default-collections.ts` 导出 `CHAT_RAG_DEFAULT_COLLECTIONS`（对话 `augmentUserTextWithRag`）与 `CONTENT_RAG_DEFAULT_COLLECTIONS`（内容创作检索）；`rag-context.ts` / `content-create.ts` 均从该文件引用，避免两处漂移。

**当前数组内容、未默认纳入的集合、本地各 `collection` 块数**：见 **`显式状态.generated.md`**（运行 `pnpm run report:explicit-state` 更新）。

| 显式说明（语义不变） |
|----------------------|
| **`wechat_style`**：已灌库可被 `retrieve` 使用，但**默认未并入**对话/创作默认列表；需在调用中显式传入 `collections` 或后续加场景开关。 |
| **`job_post`**：岗位实验数据，**不参与**对话与创作默认 RAG。 |

**前置条件**：`OPENAI_API_KEY` 存在且 **`data/knowledge/rag-store.json`** 存在时，`augmentUserTextWithRag` / 创作流检索才会执行；否则**静默跳过**检索、不报错断流。

## 三、向量块元数据（`TextChunk` / 灌库约定）

| 字段 | 适用集合 | 含义 |
|------|----------|------|
| **`collection`** | 全部 | `job_post` \| `platform_tone` \| `medical` \| `personal` \| `literature` \| `wechat_style` |
| **`paperId` / `sourceUrl`** | 主要为 `literature` | 文献主键（常为 DOI）与来源链接 |
| **`wechatStyleVariant`** | `wechat_style` | `medsci`（梅斯学术系）\| `liangyi_hui`（良医汇系） |
| **`wechatContentSlot`** | `wechat_style` | `body` \| `caption` \| `diversion` \| `references` \| `byline` \| `footer`（槽位检索） |
| **`kbWechatId`** | `wechat_style` | 与清洗稿 YAML `kb_wechat_id` 一致，便于去重 |

## 四、运行时本地文件（`data/`，默认 gitignore）

| 路径 | 作用 |
|------|------|
| **`data/knowledge/rag-store.json`** | 统一向量库；多 `collection` 合并存储。 |
| **`data/memory.json`** | **`MemoryStore`**：公司画像 **`companyProfile`**、**`contentPreferences`**（可选）、**`interactionCount`**、**`lastDeliveredDoc`**（最近成稿 docx）等（`memory/store.ts`）。 |
| **`data/chat-sessions/*.json`** | 按飞书会话 **`chatId`** 持久化：**滑动窗口** `messages` + **滚动摘要** `summary`（`memory/chat-session.ts`）；进程重启后恢复。 |
| **`data/uploads/`** | 飞书用户上传附件本地副本等（若有）。 |

## 五、三层会话记忆（与 `router` / `memory` 对齐）

| 层级 | 落点 | 作用 |
|------|------|------|
| **结构化记忆** | **`data/memory.json`** | 少而硬：公司画像、**`lastDeliveredDoc`**、**`contentPreferences`**；服务重启与路由（成稿/改稿/偏好）；经 **`formatStructuredMemoryBlock`** 注入 **system**。 |
| **滚动摘要** | **`data/chat-sessions/<id>.json`** 的 **`summary`** | 窗口溢出前若干轮经 **Claude** 压缩，控制中等 token；与窗口衔接。 |
| **滑动窗口** | 同上文件的 **`messages`** | 最近轮次原文；上限由 **`CHAT_WINDOW_*`** 等约束；每回合 **`persistChatSession`** 落盘。 |

**说明**：摘要合并会额外调用 Claude，长对话下成本与延迟可能上升；可调小窗口或提高阈值。

## 六、微信侧车（`side-tools/wechat-article-cleanup/`）

| 状态/文件 | 作用 |
|-----------|------|
| **`links.txt`** | 待抓取 URL 列表；解析器**去重**。 |
| **`links-liangyi-deferred.txt`** | **良医汇**（`mp_name` 含「良医汇」）抓取时**搁置**的 URL 记录，不入 `inbox`。 |
| **`inbox/*.raw.html`** | 抓取结果 + HTTP 注释头；清洗成功后可重命名为与 `out` 同基名。 |
| **`out/*.md`** | 清洗成品；YAML 含 `kb_wechat_id`、`wechat_style_variant` 等。 |

**流水线 CLI 开关**（`pnpm run wechat-article-pipeline`）：

| 参数 | 作用 |
|------|------|
| **`--fetch-only`** | 只抓取，不清洗。 |
| **`--clean-only`** | 只根据 `inbox` 生成 `out`。 |
| **`--strip-footer`** | 裁掉微信页尾部少量壳子提示。 |
| **`--fetch-stats`** | 尝试拉阅读/点赞等（需 **`WECHAT_MP_COOKIE`**）。 |
| **`--no-liangyi-skip`** | **不**按良医汇搁置（调试或标杆确立后强制处理）。 |

## 七、灌库脚本与「替换」行为

| 命令 | 替换的集合 | 说明 |
|------|------------|------|
| **`pnpm run ingest:presets`** | `platform_tone` + `medical`（与实现一致） | 预置 Markdown。 |
| **`pnpm run ingest:jobs`** | `job_post` | 岗位描述。 |
| **`pnpm run ingest:literature`** | **`literature` 全部** | 文献 inbox 当前全集即要保留的全集。 |
| **`pnpm run ingest:wechat`** | **`wechat_style` 全部** | 默认读微信 `out/*.md`。 |
| **`pnpm run ingest:all`** | 预置 + 岗位 | **不含**文献与微信；需分别执行 `ingest:literature` / `ingest:wechat`。 |

## 八、内容创作工作流中的「隐式」行为（便于对照）

| 状态/概念 | 说明 |
|-----------|------|
| **`GATE: READY` / `GATE: ASK`** | 动笔前核对闸门输出；解析见 `content-create.ts`。 |
| **`skipAlignmentGates: true`** | 交互卡片提交等路径**跳过**闸门与多轮追问（`content-create-card-flow.ts`）。 |
| **`FEISHU_CONTENT_FORCE_GATE=1`** | 写作简报路径上**放松/绕过**部分闸门逻辑（见 `write-brief.ts`）。 |
