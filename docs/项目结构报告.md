| 文件 | 功能 |
|------|------|
| `package.json` | 包名 `bioagent`、ESM、`scripts`：`dev` / `start` / `build` / `lint` / `probe:feishu` / **`ingest:presets`** / **`ingest:jobs`** / **`ingest:all`** / **`search:rag`** / **`search:jobs`**；依赖含 **`@anthropic-ai/sdk`**、**`@larksuiteoapi/node-sdk`**、**`openai`**（Embedding）、`dotenv`、`tsx`、`typescript`；`pnpm.onlyBuiltDependencies` 含 `esbuild`、`protobufjs`。 |
| `pnpm-lock.yaml` | pnpm 依赖锁定。 |

## 三、`src/` — 运行时源码

**作用**：飞书长连接收消息 →（目标：**RAG 检索**）→ Claude 生成 → 飞书 HTTP 发消息。  
**当前**：**里程碑 1 对话闭环**已具备；**RAG** 含统一 `rag-store.json`、`retrieve`、**用户上传文件 → `personal`**；**文本对话**经 `router` 前会 **按问句检索** `platform_tone` / `medical` / `personal`（**不含 `job_post`**，避免岗位实验干扰）；**离线** `ingest:all` 灌预置 + 岗位。

### 3.1 `src/index.ts`

1. `dotenv/config` 加载 `.env`。  
2. `validateEnv()`：要求 `ANTHROPIC_API_KEY`、`FEISHU_APP_ID`、`FEISHU_APP_SECRET`。  
3. `acquireFeishuWsLock()`（`ws-lock.ts`），防本机双开长连接。  
4. `LarkEventBridge` 监听 `message`：`messageId` 去重 → **若含附件** → `handleUserFileMessage`（`file-upload-handler.ts`）→ 文本回复；**否则** → `handleUserMessage`（`router`）→ `replyMessage` / `sendMessage`。

### 3.2 `src/lark/` — 飞书接入

| 文件 | 功能 |
|------|------|
| `events.ts` | `LarkEventBridge`：解析 `message.content` JSON；支持 **`file_key` / `file_name` → `attachment`**；无文本但有附件时仍派发事件。 |
| `feishu-client.ts` | 单例 `Lark.Client`（`appId` + `appSecret`）。 |
| `messages.ts` | `im.v1.message.create` / `reply`；当前正文为 **文本** `msg_type: text`。 |
| `message-resource.ts` | `im.v1.messageResource.get`：下载消息内用户文件到本地路径（`type=file`）。 |
| `file-upload-handler.ts` | 下载至 `data/uploads/` → `extractPlainTextFromFile`（**MVP：.txt/.md 等**）→ `ingestPersonalPlainText` 追加 **`personal`** 块。 |
| `ws-lock.ts` | 锁文件 `data/.feishu-ws-subscriber.lock`（PID），防多进程抢事件。 |

### 3.3 `src/agent/`

| 文件 | 功能 |
|------|------|
| `router.ts` | 会话历史；用户句先经 **`augmentUserTextWithRag`** 再入历史；**Anthropic** `messages.create`；正则抽取公司信息写入 `memory`。 |
| `rag-context.ts` | `augmentUserTextWithRag`：对 `platform_tone` / `medical` / `personal` 做 `retrieve`（需 `OPENAI_API_KEY` 与 `rag-store.json`）。 |

### 3.4 `src/prompts/`

| 文件 | 功能 |
|------|------|
| `medical-editor.ts` | `SYSTEM_PROMPT`、`buildSystemPrompt(companyProfile?)`，医学编辑角色与意图边界。 |

### 3.5 `src/memory/`

| 文件 | 功能 |
|------|------|
| `store.ts` | `data/memory.json`：公司画像、`interactionCount`、`lastActiveAt`。 |

### 3.6 `src/tools/`

**占位目录**，预留给云文档、多维表格等工具封装；**当前无文件**。

### 3.7 `src/knowledge/` — RAG（已建模块，里程碑 2 持续扩展）

**作用**：文本分块、**OpenAI 兼容 Embedding**（`OPENAI_*`）、**统一 JSON 向量库**、按集合合并灌库、查询嵌入与 **topK 检索**。  
**统一存储**：默认 **`data/knowledge/rag-store.json`**（多集合合并；**`personal`** 由飞书附件追加块）。

| 文件 | 功能 |
|------|------|
| `types.ts` | `KnowledgeCollection`（`job_post` / `platform_tone` / `medical` / `personal`）、`TextChunk`、`StoredVectorChunk`、`VectorStoreFile`。 |
| `collections.ts` | 导出 `COLLECTION_*`（含 **`COLLECTION_PERSONAL`**）。 |
| `paths.ts` | `PROJECT_ROOT`、`DEFAULT_RAG_STORE_PATH`、`LEGACY_JOB_POSTS_STORE_PATH`（旧版仅职位文件名，供迁移合并）。 |
| `chunk.ts` | `chunkText()`：固定窗口 + 重叠切分。 |
| `embeddings.ts` | `createEmbeddingClient()`、`getEmbeddingModel()`、`embedTexts()`。 |
| `cosine.ts` | `cosineSimilarity()`。 |
| `vector-file-store.ts` | `loadVectorStore` / `saveVectorStore`、`searchVectorStore`。 |
| `store-merge.ts` | `mergeStoreByReplacingCollections`；**`mergeStoreAppendChunks`**（追加块，用于 **personal** 多次上传）。 |
| `extract-text.ts` | `extractPlainTextFromFile`：MVP 仅 **`.txt` / `.md` 等** UTF-8 文本。 |
| `personal-ingest.ts` | `ingestPersonalPlainText`：切块嵌入并 **append** 至 `rag-store.json`。 |
| `ingest.ts` | `embedTextChunks`、`loadRagStoreOrLegacy`、`buildMergedStore`。 |
| `retrieve.ts` | `retrieve(client, query, { collections?, topK, storePath? })`。 |
| `index.ts` | 聚合导出。 |

## 五、`scripts/`（项目根）

**作用**：`tsx` 直接运行，**不进入** `tsc` 的 `rootDir` 输出；可 `import` 自 `src/knowledge/`。

| 文件 | 功能 |

## 十、核心数据流

**A. 飞书对话（当前主路径）**

```
用户文本 → WSClient（events.ts）→ index 去重 → router（rag-context 检索片段 + Claude + memory）
        → messages 回复 → 飞书

用户文件 → events 解析 attachment → file-upload-handler（下载 → 抽取文本 → personal 入库）→ 文本回复说明
```

**B. RAG 离线实验（不经过飞书）**

```
rag-presets/*.md → ingest:presets → 合并入 data/knowledge/rag-store.json（platform_tone / medical）
research/…/01-57-*.md → ingest:jobs → 合并入同一 rag-store.json（job_post）
用户查询 → search:rag / search:jobs → retrieve() → 终端展示命中块
```

**C. 目标（里程碑 2+）**

在 **`router` / 工作流** 中调用 **`retrieve`**，再 `messages.create`；`personal` 入库与飞书文档交付待接。
