# bioagent 项目结构报告

> 版本：与仓库当前结构一致（不含 `node_modules`）  
> 说明：`data/`、`dist/` 等受 `.gitignore` 约束，协作克隆后可能不存在，文中注明「运行时生成」。

---

## 一、根目录（`/`）

**作用**：项目根，放置配置、锁文件、入口说明与环境模板；产品代码在 `src/`，调研资产在 `research/`。

| 文件 | 功能 |
|------|------|
| `package.json` | 定义包名 `bioagent`、ESM、`scripts`（`dev`/`start`/`build`/`lint`/`probe:feishu`）、运行时依赖（Anthropic SDK、飞书 SDK、`dotenv`、`tsx`、`typescript`）及 `pnpm.onlyBuiltDependencies`（允许 `esbuild`、`protobufjs` 等构建脚本，保障飞书 WebSocket 编解码）。 |
| `pnpm-lock.yaml` | pnpm 依赖锁定。 |
| `pnpm-workspace.yaml` | pnpm 工作区级配置（当前含 `ignoredBuiltDependencies`，与 `package.json` 内 `pnpm.onlyBuiltDependencies` 共同约束依赖包的 postinstall 等行为）。 |
| `tsconfig.json` | TypeScript 编译选项：`ES2022`、`moduleResolution: bundler`、输出 `dist/`、根目录 `src/`、`paths` 别名 `@/*`，**排除** `research/`、`venv` 以免误编数据脚本。 |
| `.gitignore` | 忽略 `.env`、`.env.local`、`node_modules/`、`dist/`、`venv/`、`data/`、IDE 目录、日志等。 |
| `.env.example` | 环境变量模板：`FEISHU_APP_ID`、`FEISHU_APP_SECRET`、`ANTHROPIC_API_KEY`、`ANTHROPIC_BASE_URL`（如 APIMart）、`ANTHROPIC_MODEL`；**不含真实密钥**。 |
| `.env` | 本地真实配置（**被 git 忽略**），运行 `pnpm start` 时由 `dotenv` 加载。 |
| `.DS_Store` | macOS 文件夹元数据（可忽略，建议不提交）。 |

## 三、`src/` — 医学编辑 AI 员工（MVP 运行时源码）

**作用**：飞书长连接收消息 → Claude 生成回复 → 飞书 HTTP API 发回；里程碑 1 闭环。

### 3.1 `src/index.ts`（程序入口）

**职责**：组装流水线与环境校验。

**逻辑要点**：

1. `dotenv/config` 加载 `.env`。  
2. `validateEnv()`：要求 `ANTHROPIC_API_KEY`、`FEISHU_APP_ID`、`FEISHU_APP_SECRET` 存在，否则退出。  
3. 打印配置提示（模型名、飞书 AppId、单连接提示、`DEBUG_LARK` / `probe:feishu` 说明）。  
4. `acquireFeishuWsLock()`：见 `ws-lock.ts`，防止本机双开长连接抢事件。  
5. 创建 `LarkEventBridge`，监听 `message`：  
   - 用 `Set` + TTL 对 `messageId` **去重**，防飞书重推。  
   - `chatId || senderId` 作为会话键交给 Agent。  
   - 调用 `handleUserMessage` 得到文本，再 `replyMessage`（优先按 `message_id` 回复）或 `sendMessage`。  
   - 异常时尝试发送固定兜底文案。

### 3.2 `src/lark/` — 飞书接入层

| 文件 | 功能 |
|------|------|
| `events.ts` | **长连接事件桥**：`LarkEventBridge` 继承 `EventEmitter`；`start()` 用 `FEISHU_APP_ID`/`SECRET` 创建 `WSClient`，注册 `im.message.receive_v1`（解析文本、过滤 `sender_type === app` 的机器人自发言、兼容 `message` 在 `event` 子对象的情况）、`im.message.message_read_v1`（空处理消警告）；`DEBUG_LARK=1` 时提升日志级别；对外 `emit("message", LarkMessageEvent)`。 |
| `feishu-client.ts` | **单例** `Lark.Client`（应用级 `appId` + `appSecret`），供发消息 API 复用租户凭证。 |
| `messages.ts` | **发消息/回复**：`sendMessage` 调 `im.v1.message.create`（`receive_id_type` 为 `chat_id` 或 `open_id`）；`replyMessage` 调 `im.v1.message.reply`；正文统一为 `msg_type: text`，`content` 为 `JSON.stringify({ text })`（接口层虽命名 `markdown` 参数，实际按文本发送）。 |
| `ws-lock.ts` | **进程锁**：在 `data/.feishu-ws-subscriber.lock` 写入 PID；若锁存在且对应进程仍存活则直接 `exit(1)`；进程退出或 SIGINT/SIGTERM 时删除锁。 |

### 3.3 `src/agent/` — 智能体

| 文件 | 功能 |
|------|------|
| `router.ts` | **对话路由与 LLM 调用**：`Map<chatId, messages[]>` 维护每会话最多约 30 轮上下文；`loadMemory()` 读公司画像注入 `buildSystemPrompt`；`Anthropic` 客户端单例，支持 `ANTHROPIC_BASE_URL`（中转）；`messages.create` 带 `system` + 历史消息；从响应中提取 `text` 块；成功后 `extractAndSaveInfo` 用**正则**从用户句子里抽取公司名/产品等写入 `saveMemory`；API 失败时 `history.pop()` 回滚本轮用户消息并返回友好错误。 |

### 3.4 `src/prompts/` — 系统提示词

| 文件 | 功能 |
|------|------|
| `medical-editor.ts` | 导出 `SYSTEM_PROMPT` 模板与 `buildSystemPrompt(companyProfile?)`：将 `{company_profile}` 替换为本地记忆或默认「尚未了解…」占位，定义医学编辑角色、合规边界、五类意图与回复风格。 |

### 3.5 `src/memory/` — 本地记忆（里程碑 1 简化版）

| 文件 | 功能 |
|------|------|
| `store.ts` | `loadMemory`/`saveMemory` 读写 `data/memory.json`（路径相对 `process.cwd()`）；结构含 `companyProfile`、`interactionCount`、`lastActiveAt`；保存时自动递增计数与时间戳；`data` 目录不存在则 `mkdirSync`。 |

### 3.6 `src/tools/`

**作用**：占位目录，预留给里程碑 2/3 的工具调用（如文档、表格），**当前无文件**。

## 四、`scripts/`（项目根下）

**作用**：与主服务分离的一次性/诊断脚本。

| 文件 | 功能 |
|------|------|
| `feishu-ws-probe.ts` | 独立启动 `WSClient`，仅注册 `im.message.receive_v1`，收到后打印 JSON 并退出；60 秒超时提示排查多连接/凭证；用于验证长连接是否到达本机（运行前需停掉 `pnpm start`）。 |

## 九、核心数据流小结（不重复上文细节）

```
飞书用户消息
  → WSClient 收 im.message.receive_v1（events.ts）
  → index.ts 去重 → router.ts（Claude + 系统提示词 + 会话历史 + memory.json）
  → messages.ts（HTTP Client 回复文本）
  → 飞书会话展示
```
