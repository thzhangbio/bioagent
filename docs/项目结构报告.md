## 一、根目录（`/`）

**作用**：配置与锁文件、环境模板；可执行产品代码在 `src/`；调研与实验数据在 `research/`；一次性脚本在 `scripts/`。

| 文件 | 功能 |
|------|------|
| `package.json` | 包名 `bioagent`、ESM、`scripts`：`dev` / `start` / `build` / `lint` / `probe:feishu` / **`ingest:presets`** / **`ingest:jobs`** / **`ingest:all`** / **`search:rag`** / **`search:jobs`**；依赖含 **`@anthropic-ai/sdk`**、**`@larksuiteoapi/node-sdk`**、**`openai`**（Embedding）、`dotenv`、`tsx`、`typescript`；`pnpm.onlyBuiltDependencies` 含 `esbuild`、`protobufjs`。 |
| `pnpm-lock.yaml` | pnpm 依赖锁定。 |
| `pnpm-workspace.yaml` | 工作区配置（如 `ignoredBuiltDependencies` 等）。 |
| `tsconfig.json` | `ES2022`、`moduleResolution: bundler`、输出 `dist/`、根目录 `src/`、`paths` 别名 `@/*`；**仅包含 `src/**/*`**；**排除** `research/`、`venv`；根目录 **`scripts/` 不在 `include` 内**，由 **`tsx` 直接执行**，不参与 `tsc` 输出目录约束。 |
| `.gitignore` | 忽略 `.env`、`.env.local`、`node_modules/`、`dist/`、`venv/`、**`data/`**、IDE 与日志等。 |
| `.env.example` | 环境变量模板：`FEISHU_*`、`ANTHROPIC_*`；**RAG**：`OPENAI_API_KEY`、`OPENAI_BASE_URL`（如 `https://api.apimart.ai/v1`）、`EMBEDDING_MODEL`（如 `text-embedding-3-large`）；**无真实密钥**。 |
| `.env` | 本地真实配置（**勿提交**），`dotenv` 在入口与脚本中加载。 |
| `.DS_Store` | macOS 文件夹元数据（可忽略，建议不提交）。 |

**说明**：仓库根目录**无** `README.md`（若后续新增，可在本节或单独文档中补充）。

## 三、`src/` — 运行时源码

**作用**：飞书长连接收消息 →（目标：**RAG 检索**）→ Claude 生成 → 飞书 HTTP 发消息。  
**当前**：**里程碑 1 对话闭环**已具备；**RAG 子系统**已具备 **`ingest` / `retrieve` 与统一向量文件**（见 §3.7），**尚未**接入 `router` 主链路；**离线灌库**含预置调性/医学 + 57 岗位实验（`pnpm run ingest:all`）。

### 3.1 `src/index.ts`

1. `dotenv/config` 加载 `.env`。  
2. `validateEnv()`：要求 `ANTHROPIC_API_KEY`、`FEISHU_APP_ID`、`FEISHU_APP_SECRET`。  
3. `acquireFeishuWsLock()`（`ws-lock.ts`），防本机双开长连接。  
4. `LarkEventBridge` 监听 `message`：`messageId` 去重 → `handleUserMessage`（`router`）→ `replyMessage` / `sendMessage`。

### 3.2 `src/lark/` — 飞书接入

| 文件 | 功能 |
|------|------|
| `events.ts` | `LarkEventBridge`：`WSClient` 注册 `im.message.receive_v1` 等；解析文本、过滤机器人自发言；`DEBUG_LARK=1` 提高日志；`emit("message", …)`。 |
| `feishu-client.ts` | 单例 `Lark.Client`（`appId` + `appSecret`）。 |
| `messages.ts` | `im.v1.message.create` / `reply`；当前正文为 **文本** `msg_type: text`。 |
| `ws-lock.ts` | 锁文件 `data/.feishu-ws-subscriber.lock`（PID），防多进程抢事件。 |

### 3.3 `src/agent/`

| 文件 | 功能 |
|------|------|
| `router.ts` | 会话历史、`loadMemory` + `buildSystemPrompt`、**Anthropic** `messages.create`；正则抽取公司信息写入 `memory`；**尚未**调用 `knowledge` 检索与工作流。 |

### 3.4 `src/prompts/`

| 文件 | 功能 |
|------|------|
| `medical-editor.ts` | `SYSTEM_PROMPT`、`buildSystemPrompt(companyProfile?)`，医学编辑角色与意图边界。 |

### 3.5 `src/memory/`

| 文件 | 功能 |
|------|------|
| `store.ts` | `data/memory.json`：公司画像、`interactionCount`、`lastActiveAt`。 |

### 3.6 `src/tools/`

**占位目录**，预留给云文档、多维表格等工具封装；**当前无文件**。

### 3.7 `src/knowledge/` — RAG（已建模块，里程碑 2 持续扩展）

**作用**：文本分块、**OpenAI 兼容 Embedding**（`OPENAI_*`）、**统一 JSON 向量库**、按集合合并灌库、查询嵌入与 **topK 检索**。  
**统一存储**：默认 **`data/knowledge/rag-store.json`**（多集合合并；**`personal`** 尚未接飞书附件入库）。

| 文件 | 功能 |
|------|------|
| `types.ts` | `KnowledgeCollection`（`job_post` / `platform_tone` / `medical` / `personal`）、`TextChunk`、`StoredVectorChunk`、`VectorStoreFile`。 |
| `collections.ts` | 导出 `COLLECTION_JOB_POST`、`COLLECTION_PLATFORM_TONE`、`COLLECTION_MEDICAL`。 |
| `paths.ts` | `PROJECT_ROOT`、`DEFAULT_RAG_STORE_PATH`、`LEGACY_JOB_POSTS_STORE_PATH`（旧版仅职位文件名，供迁移合并）。 |
| `chunk.ts` | `chunkText()`：固定窗口 + 重叠切分。 |
| `embeddings.ts` | `createEmbeddingClient()`、`getEmbeddingModel()`、`embedTexts()`。 |
| `cosine.ts` | `cosineSimilarity()`。 |
| `vector-file-store.ts` | `loadVectorStore` / `saveVectorStore`、`searchVectorStore`。 |
| `store-merge.ts` | `mergeStoreByReplacingCollections`：按集合整类替换后与其它集合合并。 |
| `ingest.ts` | `embedTextChunks`、`loadRagStoreOrLegacy`、`buildMergedStore`（灌库前合并逻辑）。 |
| `retrieve.ts` | `retrieve(client, query, { collections?, topK, storePath? })`：查询嵌入 + 可选集合过滤 + topK。 |
| `index.ts` | 聚合导出。 |

## 五、`scripts/`（项目根）

**作用**：`tsx` 直接运行，**不进入** `tsc` 的 `rootDir` 输出；可 `import` 自 `src/knowledge/`。

| 文件 | 功能 |
|------|------|
| `feishu-ws-probe.ts` | 独立 `WSClient` 探活：`im.message.receive_v1`。 |
| `ingest-presets.ts` | 读取 `rag-presets/*.md` → 嵌入 → **替换**库内 `platform_tone`+`medical` 后合并保存；`pnpm run ingest:presets`。 |
| `ingest-job-posts.ts` | 读取 `research/…/01-57-*.md` → 嵌入 → **替换**库内 `job_post` 后合并；`pnpm run ingest:jobs`。 |
| `search-rag.ts` | 调用 `retrieve` 检索 **`rag-store.json`**；支持 `--only job_post,platform_tone` 等；`pnpm run search:rag`。 |
| `search-job-posts.ts` | 仅 `collections: [job_post]` 的封装检索；`pnpm run search:jobs`。 |

**推荐一键灌库**：`pnpm run ingest:all`（先预置，再岗位）。

## 十、核心数据流

**A. 飞书对话（当前主路径）**

```
用户消息 → WSClient（events.ts）→ index 去重 → router（Claude + memory + 提示词）
       → messages 回复文本 → 飞书
```

**B. RAG 离线实验（不经过飞书）**

```
rag-presets/*.md → ingest:presets → 合并入 data/knowledge/rag-store.json（platform_tone / medical）
research/…/01-57-*.md → ingest:jobs → 合并入同一 rag-store.json（job_post）
用户查询 → search:rag / search:jobs → retrieve() → 终端展示命中块
```

**C. 目标（里程碑 2+）**

在 **`router` / 工作流** 中调用 **`retrieve`**，再 `messages.create`；`personal` 入库与飞书文档交付待接。
