# bioagent 项目结构报告

> **版本说明**：本文档随仓库 **全量核对** 更新（2026-04-02）；描述**已存在**的路径与文件职责；不含 `node_modules`、不展开 `.git` 与 `venv`。  
> **运行时目录**：`data/`、`dist/` 受 `.gitignore` 约束，克隆后可能为空，文中标注「本地生成」。

## 一、根目录（`/`）

**作用**：配置与锁文件、环境模板；可执行产品代码在 `src/`；调研与实验数据在 `research/`；一次性脚本在 `scripts/`。

| 文件 | 功能 |
|------|------|
| `package.json` | 包名 `bioagent`、ESM、`scripts`（`dev` / `start` / `build` / `lint` / `probe:feishu` / **`ingest:jobs`** / **`search:jobs`**）；依赖含 **`@anthropic-ai/sdk`**、**`@larksuiteoapi/node-sdk`**、**`openai`**（Embedding）、`dotenv`、`tsx`、`typescript`；`pnpm.onlyBuiltDependencies` 含 `esbuild`、`protobufjs`（飞书 WebSocket 相关依赖构建）。 |
| `pnpm-lock.yaml` | pnpm 依赖锁定。 |
| `pnpm-workspace.yaml` | 工作区配置（如 `ignoredBuiltDependencies` 等）。 |
| `tsconfig.json` | `ES2022`、`moduleResolution: bundler`、输出 `dist/`、根目录 `src/`、`paths` 别名 `@/*`；**仅包含 `src/**/*`**；**排除** `research/`、`venv`；根目录 **`scripts/` 不在 `include` 内**，由 **`tsx` 直接执行**，不参与 `tsc` 输出目录约束。 |
| `.gitignore` | 忽略 `.env`、`.env.local`、`node_modules/`、`dist/`、`venv/`、**`data/`**、IDE 与日志等。 |
| `.env.example` | 环境变量模板：`FEISHU_*`、`ANTHROPIC_*`；**RAG**：`OPENAI_API_KEY`、`OPENAI_BASE_URL`（如 `https://api.apimart.ai/v1`）、`EMBEDDING_MODEL`（如 `text-embedding-3-large`）；**无真实密钥**。 |
| `.env` | 本地真实配置（**勿提交**），`dotenv` 在入口与脚本中加载。 |
| `.DS_Store` | macOS 文件夹元数据（可忽略，建议不提交）。 |

**说明**：仓库根目录**无** `README.md`（若后续新增，可在本节或单独文档中补充）。

## 三、`src/` — 运行时源码

**作用**：飞书长连接收消息 →（目标：**RAG 检索**）→ Claude 生成 → 飞书 HTTP 发消息。  
**当前**：**里程碑 1 对话闭环**已具备；**RAG 子系统已部分落地**（见 §3.7），**尚未**接入 `router` 主链路；**职位实验向量库**由 `scripts/` 离线灌库与检索。

### 3.1 `src/index.ts`

1. `dotenv/config` 加载 `.env`。  
2. `validateEnv()`：要求 `ANTHROPIC_API_KEY`、`FEISHU_APP_ID`、`FEISHU_APP_SECRET`。  
3. `acquireFeishuWsLock()`（`ws-lock.ts`），防本机双开长连接。  
4. `LarkEventBridge` 监听 `message`：`messageId` 去重 → `handleUserMessage`（`router`）→ `replyMessage` / `sendMessage`。

### 3.2 `src/lark/` — 飞书接入

| 文件 | 功能 |
|------|------|
| `events.ts` | `LarkEventBridge`：`WSClient` 注册 `im.message.receive_v1` 等；解析文本、过滤机器人自发言；`DEBUG_LARK=1` 提高日志；`emit("message", …)`。 |
| `feishu-client.ts` | 单例 `Lark.Client`（`appId` + `appSecret`）。 |
| `messages.ts` | `im.v1.message.create` / `reply`；当前正文为 **文本** `msg_type: text`。 |
| `ws-lock.ts` | 锁文件 `data/.feishu-ws-subscriber.lock`（PID），防多进程抢事件。 |

### 3.3 `src/agent/`

| 文件 | 功能 |
|------|------|
| `router.ts` | 会话历史、`loadMemory` + `buildSystemPrompt`、**Anthropic** `messages.create`；正则抽取公司信息写入 `memory`；**尚未**调用 `knowledge` 检索与工作流。 |

### 3.4 `src/prompts/`

| 文件 | 功能 |
|------|------|
| `medical-editor.ts` | `SYSTEM_PROMPT`、`buildSystemPrompt(companyProfile?)`，医学编辑角色与意图边界。 |

### 3.5 `src/memory/`

| 文件 | 功能 |
|------|------|
| `store.ts` | `data/memory.json`：公司画像、`interactionCount`、`lastActiveAt`。 |

### 3.6 `src/tools/`

**占位目录**，预留给云文档、多维表格等工具封装；**当前无文件**。

### 3.7 `src/knowledge/` — RAG（已建模块，里程碑 2 持续扩展）

**作用**：文本分块、**OpenAI 兼容 Embedding**（APIMart `OPENAI_*`）、**本地 JSON 向量文件**存取、余弦相似度检索；集合类型见 `types.ts`。  
**说明**：统一命名的 **`ingest.ts` / `retrieve.ts`** 尚未抽取；当前由下列模块 + `scripts/` 组合完成实验；**`platform_tone` / `medical` 预置库**尚未灌入，**`personal`** 尚未接飞书附件。

| 文件 | 功能 |
|------|------|
| `types.ts` | `KnowledgeCollection`（含 `job_post`、`platform_tone`、`medical`、`personal`）、`TextChunk` / `StoredVectorChunk` / `VectorStoreFile`。 |
| `collections.ts` | 导出 **`COLLECTION_JOB_POST`**（57 岗实验库集合名）。 |
| `chunk.ts` | `chunkText()`：固定窗口 + 重叠切分。 |
| `embeddings.ts` | `createEmbeddingClient()`（`openai` SDK + `OPENAI_API_KEY` / `OPENAI_BASE_URL`）、`getEmbeddingModel()`、`embedTexts()` 批量嵌入。 |
| `cosine.ts` | `cosineSimilarity()` 向量相似度。 |
| `vector-file-store.ts` | `loadVectorStore` / `saveVectorStore`（JSON）、`searchVectorStore`（全库余弦 + topK）。 |
| `index.ts` | 对上述符号的聚合导出。 |

## 四、`scripts/`（项目根）

**作用**：`tsx` 直接运行，**不进入** `tsc` 的 `rootDir` 输出；可 `import` 自 `src/knowledge/`。

| 文件 | 功能 |
|------|------|
| `feishu-ws-probe.ts` | 独立 `WSClient` 探活：是否收到 `im.message.receive_v1`（排查多连接/凭证）。 |
| `ingest-job-posts.ts` | 读取 `research/医学编辑岗-苏州-整理/` 下 **`01-`～`57-` 编号 `*.md`**，切块 → Embedding → 写入 **`data/knowledge/job-posts.json`**；`pnpm run ingest:jobs`。 |
| `search-job-posts.ts` | 加载上述 JSON，对查询句嵌入后 **topK 检索**；`pnpm run search:jobs -- "查询句"`（参数中的 `--` 会被过滤）。 |

## 九、核心数据流

**A. 飞书对话（当前主路径）**

```
用户消息 → WSClient（events.ts）→ index 去重 → router（Claude + memory + 提示词）
       → messages 回复文本 → 飞书
```

**B. 职位向量实验（离线，不经过飞书）**

```
research/…/01-57-*.md → scripts/ingest-job-posts.ts → Embedding API → data/knowledge/job-posts.json
用户查询句 → scripts/search-job-posts.ts → 嵌入查询向量 → 余弦检索 → 终端展示命中块
```

**C. 目标（里程碑 2+，与《产品蓝图》一致）**

在调用 LLM 生成前插入 **`retrieve`（多集合）**；个人素材经 **ingest** 进入 `personal`；与飞书文档工作流对接（尚未实现）。
