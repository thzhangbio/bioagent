# Formula Fragment Handbook

本文件记录 `09-formula-fragments` 在真实文稿处理中积累的经验，重点沉淀：

- 高价值、可复用的结构化规则
- 容易误判的疑难项
- 每轮处理时的审计变化
- 适合后续继续扩展的观察点

## 使用方式

- 每完成一轮明显有价值的碎片处理，就在文末追加一节
- 优先记录“模式”和“判断依据”，不要只记单篇文章的偶然字符串
- 对疑难项要同时记录：
  - 原始碎片长什么样
  - 为什么难
  - 最终怎么处理
  - 有没有残余风险

## 2026-04-06 - 第一轮疑难碎片清零

- 背景：对既有 `out/` 成稿做 `$...$` 审计后，残留从 `91/99` 连续压缩到最后 7 个疑难片段，再继续清零。
- 审计变化：最终从 `24/24` 降到 `0/0`。
- 本轮高频模式：
  - 单位和区间类残片，例如 `μ/mL`、`g/mL`、`mM β`
  - 突变位点类残片，例如 `c.748 G>A`、`c.922 A>G`
  - 化学式和缓冲液成分类残片，例如 `MgSO₄`、`Na₂HPO₄`、`KH₂PO₄`
- 新增规则：
  - 对 `\mathrm`、`\mathsf`、`\sf` 嵌套字体做更宽松匹配，避免因为 OCR 多一层字体壳导致规则漏掉
  - 对 `\\ \\mathsf`、`\\ \\mu` 这类“双反斜杠 + 空格”形态单独兜底
  - 把看似单个碎片的问题拆成多个独立片段处理，例如化学式组合行不要只匹配整串，也要支持单个 `MgSO₄`
- 疑难项：
  - `$( 1.060 - 1.100 \pm 0.01 \ : \mathrm { g / m L) }$`
  - 难点：它不是普通单位，而是“括号未闭合 + 区间 + ± + 单位”混在一个短块里
  - 处理：归一为 `(1.060-1.100 ± 0.01 g/mL)`
  - 风险：这类碎片常夹带外围多余右括号，规则要只处理片段本身，不要顺手吞上下文括号
- 疑难项：
  - `${ > } 22.2 \ \mathsf { m M } _ { \beta }$`
  - 难点：既有比较符，又有单位，又带下标 `β`
  - 处理：归一为 `> 22.2 mM β`
  - 风险：这类模式容易和真正需要保留数学语义的式子混淆，所以只在明显短片段里处理
- 疑难项：
  - `${ \sf M g S O _ { 4 } }$, 1 mM ${ \mathsf { N a } } _ { 2 } { \mathsf { H P O } } _ { 4 }$, 1.2 mM ${ \sf K H } _ { 2 } { \sf P O } _ { 4 }$`
  - 难点：一行里串了多个碎片，既像整体模式，又包含多个可独立处理的成分
  - 处理：最终归一为 `MgSO₄, 1 mM Na₂HPO₄, 1.2 mM KH₂PO₄`
  - 经验：对这种行，优先补“单片段规则 + 组合规则”两层，而不是只押注整串正则
- 仍需观察：
  - 单位 `μ/mL` 目前是按文本清洗目标保留；如果后续知识库消费更偏学术排版，可以再评估是否要统一到更自然的 `μL/mL` 或原文单位语义

## 2026-04-06 - 两篇新文稿的高频碎片批量清理

- 背景：两篇新稿进入 `out/` 后立即跑审计，初始残留为 `67/152`，高频重复明显。
- 审计变化：从 `67/152` 逐轮压到 `28/29`，再压到 `9/10`，最后清到 `0/0`。
- 本轮高频模式：
  - 引物端点：`$5^{\prime}$`、`$3^{\prime}$`
  - 倍率和成像标注：`$\times 200$`、`$\times 600$`
  - 细胞标记：`CD45+`、`CD68+`、`CD20+B`
  - 统计与指标：`p = ...`、`q = ...`、`FDR = ...`
  - 常见环境与单位：`CO₂`、`-80°C`、`48-72 h`、`500 nm`
- 新增规则：
  - 把 `3′/5′` 这种高频 qPCR/引物方向符号纳入通用规则，而不是单篇特例
  - 增加对 `log2` / `log10` / `log2(TMM + 1)` 这类常见转录组表达式的短块处理
  - 对 `CD`、`Ki67`、`FDR` 一类 biomedical token，优先按“字母数字拼合 + 正负号”文本模式处理
  - 对 `n = 8/44`、`n { = } 37 _ { \mathrm { ~ , ~ } }` 这类样本量写法加入统计类兜底
- 疑难项：
  - `$^{1,2,\#}$`
  - 难点：这是作者/单位/通讯标记类上标，不属于正文语义，也不完全等于普通文献上标
  - 处理：最后归一为纯文本上标形式 `^1,2,#`
  - 风险：后续若在作者名单区域统一做结构化抽取，这类符号可能更适合进入作者元信息层，而不是正文层
- 疑难项：
  - `$\mathrm { C D } 20 ^ { + } B$`、`$\mathrm { C D 20 + B }$`
  - 难点：`B` 细胞标记被卷进同一个公式壳里，容易只转成 `CD20+`
  - 处理：归一为 `CD20+B`
  - 经验：带尾字母的免疫标记不能只用 `CD\d+\+` 收口，要允许紧跟一个亚群字母
- 疑难项：
  - `$(+ / -$`、`$(\mathrm{AB}) + \mathrm{MO}$`
  - 难点：这种片段通常是正文括号被切裂，局部看起来不像完整语义块
  - 处理：分别按 `(+/-`、`(AB) + MO` 文本残片收口
  - 经验：对于“明显是正文切裂，而不是数学表达式”的短块，可以优先向普通文本归一，而不是保留 LaTeX 壳
- 疑难项：
  - `$\mathrm { x d } 070103 @$`
  - 难点：单看碎片像 OCR 噪点，但放回上下文后其实是 PRIDE 数据集共享邮箱的一部分
  - 处理：结合正文上下文，把 `reviewer_p xd070103@ ebi.ac.uk` 归一回 `reviewer_pxd070103@ebi.ac.uk`
  - 经验：遇到看似乱码的 token，如果它紧邻 `username:`、`password:`、数据集编号、网址或邮箱域名，要优先按“凭据 / 标识符被打散”排查，而不是直接判为噪点
- 仍需观察：
  - 免疫标记串和流式标记串正在变多，后续可以考虑从零散 regex 升级成“marker token normalizer”
  - `log`、上标、统计缩写已经开始形成一类独立子问题，未来也许值得从 `tryParenAndStatFragmentsToPlain` 中进一步拆层
