# 高质量向量化原材料判定标准

用途：
- 定义 `segment-inbox-to-out` 产出的 `out/*.kb.md` 何时可以视为“高质量向量化入知识库的标准原材料”。
- 作为处理完成后的统一验收清单，避免每次凭感觉判断。

适用范围：
- 主要适用于 `out/*.kb.md`
- 不适用于 `kb-fragment-audit.md`、`kb-fragment-list.md` 这类审计辅助文件

## 判定结论

结论分三级：
- `达标`
  - 可以直接进入知识库 ingest，通常不需要再做人手清洗。
- `接近达标`
  - 主体已可入库，但仍有会影响 chunk 质量、召回质量或可读性的局部噪声，建议再清一轮。
- `不达标`
  - 仍存在明显结构噪声、标签残留、页脚/模板污染或大面积黏连，直接入库会明显拖低质量。

## 硬性达标条件

以下条件应全部满足，才可判为 `达标`：

1. 标签与排版残留
- 不应残留原始 HTML 标签，如 `<sup>`、`<sub>`、`<h4>`、`<span>`。
- 不应出现明显 publisher / PDF 导出标签壳。

2. 公式碎片
- `kb-fragment-audit.md` 应为 `未解析种类: 0，未解析总出现: 0`。
- 若存在残留 `$...$` 但内容是合法、必要、完整的公式展示，需人工确认其不会污染向量化。

3. 页眉页脚与出版模板噪声
- 不应残留明显期刊页眉页脚，如刊名、卷期页码、`Continued on next page`、版权模板、reporting summary 模板行。
- 不应残留 `Cite this article as`、`Reprints and permissions`、`Transparent Peer Review` 这类出版流程文本。

4. 元数据纯净度
- YAML 中的 `title`、`authors`、`journal`、`doi`、`published`、`abstract` 应为纯文本。
- `abstract` 不应含 HTML 标签，不应混入无关模板文本。

5. 正文结构可切块
- 标题层级、段落边界、基本换行应保持稳定。
- 不应出现大面积“整页被揉成一段”的黏连正文。

## 软性质量条件

以下条件即使不完全满足，也未必直接判为 `不达标`，但会决定是 `达标` 还是 `接近达标`：

1. 表格块质量
- `[表格]` 可以保留，但不应长到把多个页面、多个小节、多个续表黏成单块。
- 表格文本若含大量 HTML 实体、翻页续表噪声或列边界完全消失，应优先继续清理。

2. 补充材料与方法学附录
- `STAR METHODS`、`KEY RESOURCES TABLE`、`Reporting Summary`、`Supplementary information` 不一定必须删除。
- 但若这些部分噪声占比过高、信息密度过低、明显拖低检索质量，应考虑降噪或分离。

3. OCR 可读性
- 允许少量不影响语义的 OCR 瑕疵。
- 若存在连续的乱码、错词、半句丢失、字符替换导致语义不稳，应继续处理。

4. HTML 实体与转义
- 少量 `&lt;`、`&#x27;` 若只出现在个别表格值中，可视为 `接近达标`。
- 若正文或 metadata 中大量出现，则不能判为 `达标`。

## 常见一票否决项

出现以下任一项，通常直接判为 `不达标`：
- 大量 HTML 标签残留
- 大量未解析公式碎片
- 大面积页眉页脚、卷期页码、出版模板串入正文
- 元数据摘要仍是 HTML 或模板拼接文本
- 正文整页被错误合并成超长大段，导致 chunk 不可控

## 当前建议的检查顺序

1. 跑碎片审计
- `pnpm run pdf-kb-fragment-audit`

2. 搜标签与模板噪声
- 搜 `<sup>`、`<sub>`、`<h4>`、`nature portfolio`、期刊页脚关键词

3. 抽查 metadata
- 重点看 YAML 的 `abstract`

4. 抽查长表格与附录块
- 看 `[表格]` 是否跨页黏连
- 看 `Reporting Summary` / `KEY RESOURCES TABLE` / `STAR METHODS` 是否噪声过重

## 放行建议

- `达标`
  - 可直接进入知识库
- `接近达标`
  - 若赶时间可入库，但建议先补一轮针对性清洗
- `不达标`
  - 不建议直接入库
